File: Documentation.txt
========================================

Files included in the submission:

-> Makefile
-> dogwashsynch.c
-> dogwashsynch.h
-> test_sem.c

To test:
1) Run "make"
2) Run "Test_Sem <number of bays> <number of dogs>

The test program will print out messages as dogs arrive, and start and finish
washing. It will also inform the tester of when DA or DB dogs are or are not
allowed into a bay to wash.

The test program runs three different instances of the dogwash system:
1) A dogwash with a random assorment of dogs of DA, DB, and DO types
2) A dogwash that will have more DB dogs than DA's to test that starvation
    does not occur.
3) A dogwash that will have more DA dogs than DB's to test that starvation
    does not occur.

Successes:
As you can see in the output below, the dogwash system takes in dogs and
processes them according to the rules of the assignment.

Limitations:
DO types dogs process much faster and therefore get to wash with less wait
time. This is not quite fair, but because we need to deal with exclusion for
DA and DB type dogs, we have been unable to avoid it.

In this semaphore implementation, the DO dog type seems to always print its
"Dog, O washing.\ Bays remaining: <#>" message late. Therefore, the output
can be confusing, but if you look closely the rules imposed by the assignment
spec are still being observed.
